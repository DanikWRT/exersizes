## FIX-1: Убрать дубли "Базовый" (всё ещё не работает!)

**Проблема:** Пользователь создаёт новое упражнение, добавляет его в план — видит "Базовый, Базовый".

**Причина:** `addExercise()` создаёт variant с `id = "${e.id}_Базовый"`, но при добавлении в план (`NewWorkoutScreen`) код возможно вызывает `upsertVariant` ещё раз с тем же именем, но Room видит другой id (если где-то uuid()) и вставляет дубль.

**Что проверить:**
1. `addExercise()` — создаёт ли он только один variant?
2. `NewWorkoutScreen` — при выборе exercise и нажатии "Базовый" — вызывается ли `addExercise` или `addVariant` повторно?
3. Проверить `Dao.kt` — `upsertVariant` — если передан entity с id который уже есть — обновляет. Но если id разный (один "eid_Базовый", другой uuid) — вставляет дубль.

**Решение:**
- Везде где создаётся "Базовый" — использовать один и тот же детерминированный id
- Или: в `Dao.kt` добавить `@Query("SELECT * FROM exercise_variants WHERE exerciseId = :eid AND name = :name LIMIT 1")` и проверять перед вставкой
